Reproducibility & Security Commands

run_ollama_benchmark_local.sh

Orchestrates a deterministic multi-model benchmark against a local Ollama instance.

bash scripts/run_ollama_benchmark_local.sh
  • Deterministic Validation: Freezes generation by setting temperature=0 and enforcing specific seeds.
  • JSONL Telemetry: Records granular per-trial telemetry, latency, and answer parsing strictly as JSON Lines arrays.
  • Hardware Telemetry: Extracts and bundles model API tags (size, quantization, parameter size) into metadata.json.

pull_models_disk_safe.sh

Automates parallel pulling of Ollama models while ensuring the machine does not run out of disk space.

bash scripts/pull_models_disk_safe.sh
  • Safety Floor Checks: Calculates available disk space before every pull and halts if a 55 GiB floor is reached.
  • Fast-path Existing: Queries Ollama's local tags endpoint to skip re-downloading existing models, saving bandwidth.

secret_scan.sh

Scans the repository for accidentally committed secrets (API keys, private keys) before allowing a push to GitHub.

bash scripts/secret_scan.sh
  • High-Speed Scanning: Uses ripgrep (rg) to scan the entire codebase concurrently.
  • Detected Patterns: UpCloud API tokens, AWS Access Key IDs, GitHub PATs, Slack tokens, and Private Key headers (RSA, EC, OPENSSH).

create-upcloud-token-current-ip.sh

Creates a temporary, strictly scoped UpCloud API token restricted to the public IP of the execution machine.

./create-upcloud-token-current-ip.sh --name ci-token --expires-in 4h
  • Zero-Trust Configuration: Automatically identifies the local machine's IPv4 and IPv6 public addresses.
  • IP Allowlisting: Appends exact CIDRs (/32 for IPv4, /128 for IPv6) to the API token scope mapping.
